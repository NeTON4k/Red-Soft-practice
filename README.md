## Тема практики
Поиск уязвимостей для СУБД «Ред База Данных» и/или ОС «РЕД ОС» с использованием открытых источников информации и средств автоматизированного тестирования.

## Структура репозитория

```
.
├── README.md              — этот файл
├── report/                — черновик и финальная версия отчёта
│   ├── report.md
│   └── screenshots/       — скриншоты, иллюстрации к отчёту
├── configs/                — конфигурационные файлы стенда (анонимизированные)
└── scan-results/           — сырые выводы сканеров и утилит (логи)
```

## Как пользоваться репозиторием (для себя)

1. Внести изменения в файлы.
2. `git status` — посмотреть, что изменилось.
3. `git add .` — подготовить изменения.
4. `git commit -m "краткое описание"` — зафиксировать.
5. `git push` — отправить на GitHub.

Делать это нужно **ежедневно** — таково требование практики.
